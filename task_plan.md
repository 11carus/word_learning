# WordFlow 项目任务计划

## 目标
使用 C++20、Qt 6 Widgets、CMake 和 SQLite 开发一款轻量级离线背单词桌面软件，实现单词管理、卡片复习、四档评价、简单复习日期调度、学习统计和本地数据持久化；在核心功能稳定后，将“文本型 PDF 阅读选词加入生词本”作为扩展功能纳入交付。

## 已做决策
| 决策 | 理由 |
|------|------|
| 课程 MVP 使用 Qt Widgets | 与 `tasks.md` 明确要求一致，4 天内更稳 |
| 使用 SQLite 本地数据库 | 与任务书一致，满足离线和持久化要求 |
| 使用 C++20、Qt 6.10.3、CMake、MinGW 13.1、Ninja | 环境已安装并验证 |
| 项目骨架先采用单 CMake 目标 `wordflow` | 当前范围适合课程 MVP，暂不拆分多库 |
| 第一版采用固定间隔调度：1/2/4/7 天 | 逻辑清晰、便于测试，适合课程设计范围 |
| PDF 阅读选词作为扩展功能 | 保留用户最初想法，同时不影响课程 MVP |
| 使用 GitHub 和功能分支管理 | 满足课程要求，并能体现开发过程 |
| GitHub 远程仓库使用 `https://github.com/11carus/word_learning.git` | 用户已提供仓库地址 |
| 规划文件放在项目根目录 | 便于跨会话恢复状态和实时记录改动 |

## 遇到的错误
| 错误 | 尝试次数 | 解决方案 |
|------|---------|---------|
| PPT 渲染工具依赖路径异常 | 1 | 直接解析 PPTX XML，完整读取任务书文字 |
| 附件首次读取出现乱码 | 1 | 使用 UTF-8 编码重新读取 |
| Windows 路径被 JavaScript 转义 | 2 | 工具命令统一使用正斜杠或无转义路径 |
| aqtinstall 首次 pip 命令超时 | 1 | 检查后确认安装实际已完成，未重复安装 |
| Qt 镜像组件查询校验文件下载失败 | 1 | 重试查询并成功获取架构和模块列表 |
| MinGW 实际目录名与预期不同 | 1 | 搜索安装目录后确认路径为 `mingw1310_64` |
| `tasks.md` 初次读取为空 | 1 | 用户重新指定文件后读取到 14456 字节内容并完成合并 |
| 项目尚未初始化 Git 仓库 | 1 | 当前先记录状态，阶段 3 初始化 Git |
| GitHub 远程仓库地址未提供 | 1 | 用户提供 `11carus/word_learning` 后已连接 origin |
